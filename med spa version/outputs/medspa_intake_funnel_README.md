## Import (both workflows)

1. n8n → **Workflows → Import from File** → pick the JSON.
2. Wire credentials:
   - Funnel: Anthropic (`httpHeaderAuth`, header `x-api-key`), `twilioApi`, `googleCalendarOAuth2Api`
     (logging disabled in this build — no Sheets needed).
   - Bot: Anthropic (`anthropicApi` on the Chat Model node) + Google Calendar only.
3. **Activate** both (bot memory/static data only persist on active production runs).
4. Twilio WhatsApp Sandbox → "When a message comes in" → `https://josesn8n.win/webhook/medspa-inbound` (POST).

## Test (production URLs)

```bash

curl -X POST https://josesn8n.win/webhook/medspa-intake -H "Content-Type: application/json" \
  -d '{"action":"qualify","name":"Maria Tester","phone":"+18135550100","email":"maria@example.com",
       "treatment_interest":"Interested in Botox for forehead lines, first time, weekday afternoons work best."}'
# → {"status":"ok","qualified":true,"verdict":"qualified","sms_reply":"…","slots":[…]}

# Step 2 — book a returned slot (iso from step 1)

# → {"status":"ok","booked":true,"consult_slot":"…","sms_reply":"…"}
```

## Compliance

The AI qualifies & schedules ONLY. No medical advice, no treatment-suitability or outcome
statements, no diagnosis, no price promises, no provider-patient relationship. Enforced in the
funnel's `Set: AI Config` node and the bot's agent system message — never weaken on rebuild.
